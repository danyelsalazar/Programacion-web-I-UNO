Entendé la diferencia de verdad

¿Qué es Git?

Git es un sistema de control de versiones distribuido. Sirve para registrar cambios en archivos a lo largo del tiempo, permitiendo:

  • Volver a versiones anteriores
  • Trabajar en equipo sin pisarse el código
  • Crear ramas (branches) para desarrollar nuevas funcionalidades

¿Qué es GitHub?

GitHub es una plataforma en la nube que utiliza Git. Permite:

  • Guardar repositorios en internet
  • Colaborar con otros desarrolladores
  • Compartir proyectos
  • Gestionar cambios mediante pull requests

GitHub necesita Git, pero Git NO necesita GitHub.

Diferencias clave entre Git y GitHub

Git GitHub
Herramienta (software) Plataforma (servicio web)
Funciona localmente Funciona en la nube
Control de versiones Hosting de repositorios
No requiere internet Requiere internet

Flujo de trabajo

  1. Crear repositorio (GitHub)
  2. Clonar repositorio (Git)
  3. Crear cambios en el código
  4. git add → preparar cambios
  5. git commit → guardar cambios
  6. git push → subir cambios a GitHub
